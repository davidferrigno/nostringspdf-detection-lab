OPEN ARCHITECTURE DECISION — make this with fresh judgment:

Option A: Copy production heuristic code from main repo to lab
   Pros: fastest path to real measurement
   Cons: code drift, lab becomes coupled to production internals,
         two copies of the same logic to maintain

Option B: Import main repo as Python path (pip install -e ../nostringspdf)
   Pros: cleanest reuse, single source of truth for heuristic
   Cons: lab dependency on main repo directory layout, breaks if
         main repo refactors, awkward when lab is on VPS but main
         repo is only on Windows

Option C: Reimplement clean lab baseline from scratch
   Pros: lab tests ideas independently, can evolve without affecting
         production, clean abstraction surface
   Cons: more work upfront (~500-800 lines), two implementations
         to keep functionally aligned

CURRENT LEAN: Option C
   The lab's purpose is to TEST IDEAS. If the lab implementation IS the
   production implementation, then "lab measurement" and "production
   change" become impossible to separate. The whole point is that the
   lab can try things production hasn't.

   Production heuristic stays where it lives (services/heuristic_service.py
   in main repo). Lab gets its own heuristic_lab_v1.py that starts as
   a clean reimplementation of the same algorithm. Future improvements
   experiment in the lab first, then port to production only when proven.

TASK 6: DETECTION COMPARISON OVERLAY RENDERER (after Task 4)
================================================================

Build render_detection_overlays.py that takes:
  - A benchmark scorecard.json
  - Source PDFs
  - Ground truth JSONs

And outputs per-page PNGs showing:
  - Ground truth fields in green (translucent)
  - Detected fields in red (translucent)
  - Overlap regions in yellow (intersection of both)
  - FP detections (red only, no green underneath) clearly visible
  - FN ground truth (green only, no red overlay) clearly visible

This becomes the "diff viewer" for benchmark runs. Once it exists, we
can immediately see WHERE the heuristic is wrong on each PDF, not just
that it's wrong by some percentage.
